# Agent Identity Protocol — 商业化策略

**日期**: 2026-03-25
**状态**: Draft
**范围**: AIP 生态运营方的商业模式与数据策略

这份文档跟 AIP 协议规格是两回事。协议是开放标准，谁都能实现；这份文档讲的是——运营 IdP 和活动追踪器的人，怎么在这个生态上建立可持续的生意。

---

## 1. 数据资产

跨平台活动图谱是 AIP 生态最核心的数据资产。它从协议的活动证明层（Layer 2）自然生长出来。

### 1.1 会积累什么数据

随着生态发展，活动追踪器会积累这些东西：

| 数据 | 来源 | 价值 |
|------|------|------|
| 智能体注册表 | 注册流程 | 谁存在、谁造的、用什么模型 |
| 认证事件 | Token 签发 | 智能体什么时候活跃、用了哪些服务 |
| 活动报告 | Hub 签名证明 | 智能体做了什么、表现如何、结果怎样 |
| 跨平台行为图谱 | 聚合分析 | 智能体在整个生态里的完整画像 |

单个 Hub 看不到全貌。只有活动追踪器能看到所有服务的数据。

### 1.2 为什么数据会越滚越大

网络效应让数据随时间指数增值：

1. 更多智能体注册 → 更丰富的画像 → 更准确的信任评分
2. 更准的信任评分 → 更多 Hub 来查询 → 更多 Hub 上报活动
3. 更多活动数据 → 更好的洞察 → 更多客户愿意买洞察
4. 更多客户 → 更多收入 → 更多投入平台建设
5. 更多 Hub 接入 → 竞争对手越来越难复制这份数据集

第一个达到临界规模的活动追踪器运营方，就拥有了智能体行为图谱。这就是护城河。

## 2. Experian 模型

这份数据资产支撑三条业务线，类比 Experian 的征信局模型：

```
┌─────────────────────────────────────────────────────┐
│                  AIP 数据资产                         │
│           （跨平台智能体活动图谱）                      │
├─────────────────┬─────────────────┬─────────────────┤
│  信任征信        │  洞察分析        │  合规审计        │
│                 │                 │                 │
│  智能体信任评分  │  脱敏聚合的      │  给监管方和      │
│  帮平台做准入    │  行为分析，卖给  │  企业治理的      │
│  决策            │  平台方和研究者  │  审计记录        │
│                 │                 │                 │
│  （信用分）      │  （数据服务商）  │  （合规报告）    │
└─────────────────┴─────────────────┴─────────────────┘
```

**信任征信** — 平台来问："我该信任这个智能体吗？"基于经过验证的活动历史来回答。就像银行放贷前查征信。

**洞察分析** — 脱敏、聚合的数据产品：市场趋势、智能体采用模式、模型性能基准。卖给平台方、研究者、投资人。

**合规审计** — 给受监管行业提供审计线索。"把智能体 X 过去90天的所有活动给我看看。" EU AI Act、金融监管、企业治理都需要这个。

## 3. Hub 上报激励

Hub 向活动追踪器上报活动数据。难点在于：上报对整个生态有利，但对单个 Hub 没有直接好处。经典的公共品问题。

### 3.1 互惠模型（核心机制）

上报活动数据的 Hub，可以查询入站智能体的信任评分和活动历史。不上报的 Hub 查不了。不允许搭便车。

这是最强的激励——Hub 想审查入站的智能体，而获取这些数据的唯一方式就是贡献自己的数据。

### 3.2 其他激励

| 激励 | 怎么运作 |
|------|---------|
| **认证徽章** | "AIP Verified Hub"——智能体更愿意选认证过的 Hub。品牌效应。 |
| **发现/列表** | 活动追踪器同时是 Hub 目录。上报活动 → 你的 Hub 能被智能体发现。 |
| **欺诈检测** | 跨平台视角能发现单个 Hub 看不到的恶意行为模式。共享情报。 |
| **合规需要** | 对受监管的 Hub 来说，活动上报可能是法律要求。 |

### 3.3 冷启动

第一天：没有 Hub 上报，没有数据，没有信任评分可以提供，Hub 也没有理由上报。

打破死循环的方式跟征信局一样——从自己人开始：

1. DojoZero 是自家 Hub，第一天就上报，提供种子数据。
2. 基于 CoPaw/OpenClaw 构建的 Hub 默认上报——写在 SDK 里。
3. 早期合作伙伴 Hub 免费用信任评分，交换条件是上报数据。
4. 等到5-10个 Hub 在上报，数据就有足够价值了，新 Hub 自然想加入。

活动追踪器不需要第一天就对所有人有用。先对自己的生态有用，再向外扩张。Experian 当年也是从几家银行互相共享数据开始的，征信局是后来的事。

## 4. 收入来源

### 4.1 信任评分 API（B2B）

Hub 按查询次数或订阅方式付费，在授权前查智能体的信任评分。

- 免费档：基础信任评分（活跃/不活跃、身份年龄）
- 付费档：详细活动历史、跨平台行为分析、风险评估

### 4.2 洞察 API（B2B）

脱敏、聚合的数据产品：

- 按框架、模型、地区划分的智能体采用趋势
- Hub 使用模式和增长指标
- 模型在真实任务中的性能基准
- 给投资人和平台方的市场情报

### 4.3 合规套餐（B2B/企业）

给受监管行业的审计线索：

- 特定智能体的完整活动历史（需开发者授权）
- 符合 EU AI Act、金融监管要求的合规报告
- 企业治理看板
- 自托管活动追踪器，满足数据主权要求

### 4.4 高级 IdP 功能（B2B/开发者）

IdP 本身免费（开放协议）。托管 IdP 的增值功能收费：

- SLA 保障（可用性、延迟）
- 高级密钥管理（HSM 托管密钥、自动轮换）
- 组织管理（团队、角色、审计日志）
- 优先技术支持

## 5. 隐私与授权

跟 Web 时代的追踪（偷偷摸摸、没人同意）不一样，AIP 的活动数据是：

- **明确的** — 开发者按智能体逐个选择加入，隐私级别清楚明白
- **结构化的** — 定义好的 schema，不是从点击流里猜出来的
- **经授权的** — 设计上就符合 GDPR 等数据保护法规
- **可控的** — 开发者随时可以调整隐私级别或删除数据

这让数据在法律上更站得住脚，准确性也比传统行为追踪高。而且形成了竞争优势——企业和监管方更偏好基于授权的数据系统。

## 6. 竞争定位

### 6.1 vs Microsoft Entra Agent ID

微软解决的是 M365 生态内的企业智能体治理。AIP 解决的是开放生态里跨框架、跨平台的身份问题。两者共存——就像 Active Directory 和"用 Google 登录"服务不同市场。

微软的入局验证了这个问题空间，让智能体身份成为行业共识。这对 AIP 有利——市场会需要一个开放的、不锁定厂商的替代方案。

### 6.2 护城河

护城河是跨平台活动图谱，不是协议本身。协议是开放的——谁都能搞一个 IdP。但数据最多的活动追踪器赢，因为：

- 信任评分的准确度取决于背后的数据量
- Hub 倾向选覆盖面最广的活动追踪器
- 智能体倾向选接入最好活动追踪器的 IdP
- 数据会复利——历史活动记录是新玩家复制不了的
